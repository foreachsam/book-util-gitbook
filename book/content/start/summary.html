GitBook Entrance

建立架構

編輯 summary

編輯「SUMMARY.md」

$ vi SUMMARY.md

原來的內容


# Summary

改為下面的內容


# Summary

* [文章一](content/article_001.md)
* [文章二](content/article_002.md)
    * [文章三](content/article_003.md)
    * [文章四](content/article_004.md)
* [文章五](content/article_005.md)
* [文章六](content/article_006.md)

產生md檔

執行

$ gitbook init

顯示

info: init book at /home/user/Project/Gitbook/demo-book 
info: detect structure from SUMMARY (if it exists) 
info: found README.md 
info: create content/article_001.md 
info: create content/article_002.md 
info: create content/article_003.md 
info: create content/article_004.md 
info: create content/article_005.md 
info: create content/article_006.md 
info: initialization is finished 

Done, without error

上面的動作，會根據「SUMMARY.md」設定，產生相關的md檔。

$ tree ./

顯示

./

│   ├── article_002.md

觀看網頁書本內容

執行下面指令，啟動「Server」

$ gitbook serve

可以看到左方的選單。

不同的流程

上面的流程是撰寫「SUMMARY.md」，然後執行「gitboo init」，來產生相關的「md檔」，執行「gitbook serve」就可以看到書本內容了。

也可以改用另一個流程。先產生「md檔」，然後再撰寫「SUMMARY.md」，執行「gitbook serve」就可以看到書本內容了。
